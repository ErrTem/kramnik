## Shared Patterns

### Root-orchestrated Docker + Prisma CLI

**Source:** `package.json` (`db:up`), `02-RESEARCH.md` Pattern 2  
**Apply to:** root `package.json`, README, verify scripts

Always start Postgres before migrate/seed:

```bash
pnpm db:up
# copy apps/api/.env.example → apps/api/.env
pnpm db:migrate
pnpm db:seed
```

Prisma commands must use workspace filter so `DATABASE_URL` resolves from `apps/api/.env`:

```bash
pnpm --filter @kramnik/api exec prisma <command>
```

### Dev-only credentials documentation

**Source:** `README.md` § Security (dev only)  
**Apply to:** README dev logins, seed passwords

Mirror Postgres tutorial-credential warning for app users: label **dev only**, never production (D-28). Same tone as docker-compose comment:

```1:1:docker-compose.yml
# Dev-only tutorial credentials (postgres/postgres, shop_dev). Never use in production.
```

### Shell verify script conventions

**Source:** `scripts/verify-postgres.sh`, `scripts/verify-phase1.sh`  
**Apply to:** `scripts/verify-phase2.*`

- Bash: `#!/usr/bin/env bash`, `set -euo pipefail`, precondition comment at top
- PowerShell: `$ErrorActionPreference = 'Stop'`, explicit exit code checks after `docker compose exec`
- Use `docker compose exec -T postgres` against service name `postgres` (matches `docker-compose.yml` lines 3-4)

### Vitest static contract tests

**Source:** `tests/foundation/postgres-scripts.test.ts`, `readme-dev-flow.test.ts`  
**Apply to:** all `tests/data/*` and `verify-phase2-scripts.test.ts`

Read committed files with `readFileSync`; substring assertions only; no Docker in unit tests unless marked integration/manual.

### Monorepo package boundaries (Phase 2)

**Source:** `.planning/PROJECT.md`, `02-CONTEXT.md`  
**Apply to:** all Phase 2 files

- No `PrismaModule`, controllers, or REST changes
- No `packages/types` Prisma generate yet (D-07 document path only)
- Schema lives in `apps/api/prisma/` only (D-01)
